Comparison of Integrity and Authenticity

Aspect Integrity Check (Hash Verification) Authenticity Check (GPG Signature Verification)
Definition Ensures that the file’s hash (e.g., SHA-256) matches a reference hash, confirming that the file has not been modified during download or transfer. Verifies the file's origin and authenticity by checking its digital signature, ensuring it was signed with the publisher's **private key** and can be verified with their **public key**.
Purpose Detects file corruption, accidental modifications, or unauthorized changes during transfer or storage. Confirms that the file is from the expected publisher and has not been tampered with, protecting against impersonation and malicious modifications.
Method Generates a checksum (hash) of the downloaded file and compares it with an official hash (typically available on the publisher’s website). Uses GPG to verify the digital signature attached to the file, checking that the signature was created using the publisher's **private key** and matches the corresponding **public key**.
Commands sha256sum file.img (check the hash) and compare it with the official reference hash provided by the publisher. gpg --verify file.sig file.img or gpg --verify file.asc file.img (verify the GPG signature using a pre-imported public key).
Protection Against Data corruption, accidental file modifications, and integrity issues during download or storage. Malicious tampering, impersonation, and supply-chain attacks by verifying that the file was genuinely signed by the expected publisher.
Limitations Does not provide any information about the source or creator of the file; it only checks for modifications to the file. Requires access to the publisher's **public key** and the ability to trust that key, meaning the key must already be imported and trusted in your GPG keyring.
File Types for Signature Hash values are typically provided as plain text (e.g., SHA-256 sums).
These are not files you verify but are manually compared.		 Digital signatures are provided as `.sig` (binary format) or `.asc` (ASCII-armored format) files. These files contain the cryptographic signature for the corresponding file.
.sig vs .asc Files Not applicable for integrity checks. Integrity checks rely on hash comparisons rather than signatures.
  • .sig: A binary format containing the cryptographic signature (used in automated systems).
  • .asc: An ASCII-armored, human-readable format of the same signature (suitable for text-based sharing or emails).
Key Concepts
  • Hashing: A cryptographic function that generates a unique checksum of a file's contents, used for integrity verification.
  • Public Key: A key used to verify signatures. It is publicly available and used to ensure that the file was signed with the corresponding private key.
  • Private Key: A secret key used to create the signature. It is kept confidential by the publisher.
  • Signature: A cryptographic value generated by the private key, proving that the file was signed by the publisher and has not been altered.
Example Usage 
sha256sum tails.img
(compare this output with the official hash on the website).		 
gpg --verify tails-amd64-6.9.img.sig tails-amd64-6.9.img
(verify the signature of the file against the publisher's public key).

Git Repository Size Management Cheat Sheet

1. Check File Sizes in the Working Directory

du -sh *

This command shows the sizes of all files and directories in the current directory.

2. Estimate Size of Changes

du -sh path/to/file

This allows you to check the size of specific files before adding them to your repository.

3. Use Git Status

git status

This command displays the current status of the working directory and staging area, helping you identify modified or untracked files.

4. Check Total Repository Size

git count-objects -vH

This provides a detailed view of the size of your repository, including the number of objects and the total size in a human-readable format.

5. Clean Up Your History

git gc

This command runs garbage collection, which helps clean up unnecessary files and optimize the local repository, potentially reducing its size, especially if you often add and remove large files.

6. Monitor Repository Size

git count-objects -vH

Re-running this command after cleanup or after significant changes will help you monitor your repository size over time.

Split Large Pushes

To avoid exceeding GitHub's size limits, break your changes into smaller, manageable commits.


git add part1
git commit -m "Commit part1"
git push origin main

git add part2
git commit -m "Commit part2"
git push origin main

Important Notes:

  • Files larger than 50 MiB will trigger a warning, while files over 100 MiB are blocked from being pushed to GitHub.
  • GitHub enforces a 2 GiB limit on the size of a single push and recommends keeping repositories under 1 GB, with a strong suggestion to stay below 5 GB for optimal performance.

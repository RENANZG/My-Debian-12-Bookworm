Git Basics: Initial Setup and Next Steps

This manual outlines the essential steps to get started with Git and provides additional instructions for more advanced operations. It is intended for users who are new to version control and wish to establish a solid foundation in Git usage.

1. Initial Repository Setup

Follow these steps to create a Git repository and push changes to a remote server:

1.1 Initialize the Repository

Begin by initializing the Git repository in your local project directory:

git init

1.2 Add a Remote Repository

Link the local repository to a remote Git hosting service (such as GitHub, GitLab, etc.) using the following command:

git remote add origin <remote_url>

Replace <remote_url> with the actual URL of your remote repository.

1.3 Stage Files for Commit

Stage all modified and new files for the next commit:

git add .

1.4 Commit the Changes

Record the staged changes with a meaningful commit message:

git commit -m "Initial commit"

1.5 Push Changes to the Remote Repository

Push the committed changes to the remote repository. For an incremental update (push only the differences), use:

git push origin main

2. Advanced Operations: Force Pushing

In cases where you need to overwrite the remote repository entirely, a force push can be used. Be cautious as this will delete any changes on the remote that are not present in the local repository.

2.1 Force Push

Use the following command to overwrite the remote repository with your local changes:

git push origin main --force

Substitute main with your branch name, if necessary.

3. Next Steps in Git Usage

Once you are comfortable with the basics of creating repositories, committing, and pushing changes, it’s important to learn additional operations that will enhance your workflow.

3.1 Cloning a Repository

To begin working on an existing project, you can clone a remote repository to your local machine. This will create a local copy of the project:

git clone <remote_url>

This command downloads the entire project history, including all branches, commits, and files.

3.2 Pulling Changes

To synchronize your local repository with the latest changes from the remote repository, use the git pull command. This operation updates your local files with any new commits from the remote branch:

git pull origin main

Ensure you resolve any merge conflicts that might arise if both your local and remote repositories have conflicting changes.

3.3 Branching

Branching allows you to create isolated environments for development, experimentation, or bug fixes without affecting the main project code. To create a new branch:

git branch <branch_name>

Switch to the new branch using:

git checkout <branch_name>

After making changes, you can merge the branch back into the main codebase. Switch to the main branch and merge the changes:

git checkout main
git merge <branch_name>

3.4 Handling Merge Conflicts

Conflicts occur when changes from different sources (e.g., local and remote repositories) overlap. Git will pause the merge and mark the conflicting sections in the files. You must manually edit these sections, choose the correct changes, and then commit the resolved files:

git add .
git commit -m "Resolved merge conflict"

3.5 Fetching Remote Updates

To check for updates in the remote repository without merging them into your local repository, you can use:

git fetch origin

This retrieves the changes but does not apply them to your local working directory until you explicitly merge them.

4. Summary of Git Commands

Here’s a brief summary of the commands covered in this manual:

  • git init – Initialize a new Git repository
  • git remote add origin <remote_url> – Link the local repository to a remote one
  • git add . – Stage files for commit
  • git commit -m "message" – Commit changes with a message
  • git push origin main – Push changes to the remote repository
  • git push origin main --force – Force push to overwrite the remote repository
  • git clone <remote_url> – Clone a remote repository
  • git pull origin main – Pull changes from the remote repository
  • git branch <branch_name> – Create a new branch
  • git checkout <branch_name> – Switch to a different branch
  • git merge <branch_name> – Merge a branch into the current branch
  • git fetch origin – Fetch updates from the remote repository
